The basic functionality of the e-basket application had to be done
using only jsp without servlet.
The interface is a table where the user has to enter the desired quantity
of 3 items,coffe,sugar and water in order to get the total amount of the price
to be paid.
The whole implementation of the application is done in a jsp file
using an html form where a table is created and jsp commands are
embedded in the correct position of the table

In this implementation of my e-basket application i have used
cookies.
I use a cookie for every one of the three user's inputs for the
quantity of each of the three items.

So i used a cookie for the quantity of the coffe the user enters,one
for the quantity of sugar,and one for the quantity of water.

So the user fills the form and hits the submit button.
If the user closes the browser and then opens it again in no more
than 5 fives,in the
quantity field of the form for every item, will be the user's
last choice entered.
So the first thing i do in my jsp is to get the cookies.
If the cookies are not null i put their value inside the form.
I set my cookies when the user has already filled the form and hitted the
submit button.
The max age of my cookies are 5 minutes.

In my implementation i have code for the form-table to be appeared in the
page of the browser if the user hasn't entered anything yet
(e.g request.getParameter("qcafe")==null
where the variable qcafe has the quantity of coffee the user has entered.)
In this situation the form-table is appeared empty or with the values of the
quantity-cookies.

I also have code for the form to be appeared in the
page of the browser if the user has entered data and submitted the form
where the form has the desired result in the right positions of the table.

I also parse the the price values from an xml file.
